# Status Files Archive

Historical status and progress files from the investigation period.

## About this archive

Many markdown files in this directory are historical status updates from during the investigation. These files document progress but are not needed for current research or complaint filing.

**You can ignore these files** - they are kept for historical reference only.

## File categories

### Status files

Files documenting investigation status:
- `PROJECT_COMPLETE.md`
- `INVESTIGATION_STATUS_COMPLETE.md`
- `FINAL_STATUS_RECONCILIATION.md`
- `VIRGINIA_EXTRACTION_FINAL_STATUS.md`
- `VIRGINIA_EXTRACTION_COMPLETE_SUMMARY.md`

### Progress files

Files documenting progress updates:

- `FINAL_DELIVERABLES_SUMMARY.md`

## What to use instead

**For current research:**
- Use `README.md` - Main research directory guide
- Use `DATA_GUIDE.md` - Complete data guide
- Use `RESEARCH_INDEX.json` - Master file index

**For complaint filing:**
- Use files in `va_dpor_complaint/` directory
- Use files in `reports/` directory

**For understanding data:**
- Use JSON files in subdirectories
- Use summary reports in `reports/` directory

## File organization

Status files are kept for historical reference but can be ignored for:
- Current research
- Complaint filing
- Data analysis
- Understanding findings

Focus on:
- JSON files (actual data)
- Files in `reports/` (summaries)
- Files in `va_dpor_complaint/` (complaint package)
- Main guides: `README.md`, `DATA_GUIDE.md`
